# JingGraph 交互式文件知识图谱索引器原型

## 概述
JingGraph 致力于通过图谱化方式追踪灵感与概念的生命过程，结合 AI 进行语义演绎与情绪评估，最终形成具有强记忆与共鸣能力的文档索引系统。视觉风格采用柔美暗色基调、曲线边框与留白艺术，辅以时间粒子与灵感浮标等元素。

## 核心功能模块

1. **灵感追踪器（Idea Lineage）**
   - 解析文档草稿中出现的关键概念或段落，记录其起点与所有后续版本。
   - 在图谱上形成 "灵感生命周期" 曲线，可回溯不同稿次的演变。
   - 提供版本对比与时间轴查看，支持 AI 生成的演化解读。

2. **概念变异映射（Semantic Evolution）**
   - 以关键词（如“信任”“召唤”“沉默”）为中心节点，收集并展示其在不同场景中的语义漂移链。
   - 实现并列视图与变体可视化，帮助用户观察语义在上下文中的流变。

3. **片段复活机制（Dormant Segment）**
   - 自动识别高情绪浓度但未被结构化的文本段落，标记为 "潜在片段"。
   - 结合 AI 分析提出重写或嵌入建议，并在图谱中以特殊标识呈现。

4. **情绪可视化轨道（Emotional Flow）**
   - 从文档中提取情绪线索（如“靠近”“崩溃”“共鸣”），按照时间构建情绪张力轨迹。
   - 用户可按角色、章节或事件查看情绪走势，并与概念/片段关联。

5. **结构漏洞雷达（Unresolved Node）**
   - 检测结构中未闭合的节点：未回应的对话、未回响的动作或象征等。
   - 提供补全或跳转建议，将待解决节点与相关段落相连。

## 数据结构与存储

- **图数据库/数据格式**：推荐使用 Neo4j、D3.js 或 Cytoscape.js。下方 JSON Graph Schema 为 v0.1 版本，可扩展为 RESTful API。

```json
{
  "nodes": [
    {"id":"node-001", "type":"concept", "label":"信任", ...},
    {"id":"node-002", "type":"emotion_segment", "label":"剪影下的自由", ...},
    {"id":"node-003", "type":"draft_fragment", "label":"未使用片段 · 她未回头", ...}
  ],
  "edges": [
    {"source":"node-001","target":"node-002","type":"emerges_as", ...},
    {"source":"node-001","target":"node-003","type":"semantic_drift", ...},
    {"source":"node-002","target":"node-003","type":"fragment_refers_to", ...}
  ]
}
```

- **扩展字段**：
  - `references`：指向原文档或云端链接
  - `AI_notes`：由 ChatGPT 生成的分析或改写提示
  - `timeline_rank`：用于时间排序与绘图展示

## AI 对话提示系统

- 针对每个节点，系统会自动生成提示，如：
  - *“请描述 node-001 （信任）在最新版本中的语义变化。”*
  - *“该潜在片段未被整合，是否需要重写或嵌入？”*
- 用户可直接在界面上与 AI 对话，生成新的图谱节点或更新关系。
- AI 将根据 JSON schema 自动填充节点属性并提供分析。

## 用户交互流程（简化）

1. **上传文稿**：用户将文本或文档上传。
2. **语义分析**：AI 解析文本，生成初步节点（概念、情绪、片段）。
3. **图谱呈现**：
   - 左侧为概念/情绪索引树，右侧为主图谱区域。
   - 用户可拖拽节点调整布局，或手动添加记忆片段与引用。
4. **灵感生命曲线**：
   - 时间轴下显示灵感起点与变体，点击可展开版本对比。
5. **情绪轨道视图**：
   - 选择角色或章节后展示情绪张力曲线。
6. **结构漏洞与复活**：
   - 雷达视图提示未闭合节点，用户可点击查看详情并接受 AI 建议。

## 前端模块建议

- **技术栈**：React / Vue + D3.js 或 Cytoscape.js；CSS 动效建议采用暗色柔光、曲线边框和留白。
- **交互元素**：
  - 主要图谱：支持节点拖拽、缩放、右键菜单（添加/编辑/删除）。
  - 时间粒子：沿时间轴缓慢移动，象征灵感流动。
  - 灵感浮标：悬浮在节点旁，展示 AI 提示或情绪强度。
- **模式布局**：
  - 顶部导航：文档管理、视图切换、AI 聊天入口。
  - 左侧栏：节点过滤与搜索；右侧栏：详细属性面板与 AI 建议。

## 数据流逻辑

1. **上传阶段**：文本上传后经后端服务（可用 Python Flask/FastAPI）解析。
2. **解析阶段**：调用 NLP/情感分析模型生成节点与边，写入图数据库或前端暂存。
3. **前端渲染**：通过 Graph API 加载节点与边，使用 D3 或 Cytoscape 绘制。
4. **交互更新**：用户的拖拽、编辑操作通过 API 回传，实时更新数据库。
5. **AI 交互**：前端与 ChatGPT 通讯，生成 `AI_notes` 或新节点，更新图谱展示。

## UI 风格示例

- 暗色柔和背景，辅以半透明曲线框，创造 "夜色诗性" 体验。
- 适度留白，节点采用圆润渐变，并根据情绪强度呈现不同色彩（如红橙表示高潮，蓝灰表示平静）。
- 时间粒子与灵感浮标在页面上缓缓流动，增加视觉层次感。

## 逻辑层模型（概念草图）

```
[文稿] → [解析] → [概念节点 / 情绪节点 / 片段节点] → [图谱展示]
   |                                             ↘
   +→ [结构漏洞雷达]←---------↙               ↘
   +→ [AI 复写/延伸建议] —→ [用户交互] —→ [节点调整或新增]
```

以上设计旨在让 JingGraph 成为一个可追溯灵感、揭示语义变异、帮助复活潜在片段并可视化情绪张力的创作伙伴。为未来开发原型提供了结构与流程参考。
